# 🎸 기타 코드 운지 교정기 — Phase 0~1 출발점

웹캠으로 손을 비추면 **손가락 21개 관절을 실시간 추적**해 화면에 골격으로 그려주는 출발점입니다.
이후 Phase 2~에서 지판 보정 → 코드 판정 → 교정 피드백을 얹습니다.

## ⚠️ 중요: 반드시 "로컬 서버"로 실행하세요
카메라(`getUserMedia`)는 **보안 컨텍스트(https 또는 localhost)** 에서만 동작합니다.
`index.html`을 더블클릭해 `file://` 로 열면 카메라가 켜지지 않습니다.

### 실행 방법 (택1)

**Python (설치돼 있으면 가장 간단)**
```bash
# 이 폴더에서
python -m http.server 8000
# 브라우저에서 http://localhost:8000 접속
```

**Node.js**
```bash
npx serve .
# 또는
npx http-server -p 8000
```

**VS Code**
- "Live Server" 확장 설치 → `index.html`에서 우클릭 → *Open with Live Server*

## 🌐 인터넷 연결 필요 (CDN)
최초 실행 시 MediaPipe 모듈·WASM·손 인식 모델을 `cdn.jsdelivr.net` 과 `storage.googleapis.com` 에서 내려받으므로
**인터넷 연결이 필요합니다.** 카메라 영상은 정상인데 상태 패널의 "모델"이 **실패**로 표시된다면,
광고 차단기·사내 프록시·지역 차단이 위 두 호스트 접근을 막고 있는지 확인하세요.
(이 CDN들은 CORS 헤더를 정상 제공하므로 CORS 설정 문제는 아닙니다.)
완전한 오프라인 사용이 필요하면 이후 단계에서 해당 자원을 버전 고정해 **자체 호스팅**하세요.

## 사용법
1. 위 방법으로 서버를 띄우고 브라우저에서 접속.
2. **카메라 시작** 클릭 → 권한 허용.
3. 손을 카메라에 비추면 골격과 손끝(1·2·3·4·T 번호), 왼/오른손 라벨이 표시됩니다.
4. 우측 패널에서 거울 모드/골격 표시 토글, 카메라 장치 선택 가능.

### 🎯 지판 보정 (Phase 2)
운지 판정을 하려면 먼저 카메라에 비친 지판을 좌표계로 등록해야 합니다.
1. 기타 지판이 카메라에 잘 보이게 두고 **카메라 시작**.
2. **기준 프렛 K**(기본 6) 확인 후 **🎯 지판 보정 시작** 클릭.
3. 화면 상단 안내대로 **4점**을 순서대로 클릭:
   1. 6번줄(굵은 줄) × 너트   2. 1번줄(얇은 줄) × 너트
   3. 1번줄 × K프렛선   4. 6번줄 × K프렛선
4. 클릭이 끝나면 **녹색 격자**가 지판 위에 그려집니다. 실제 지판과 잘 겹치면 성공.
   (프렛 간격은 평균율 공식으로 자동 배치 — 고프렛일수록 좁아짐)
5. 보정값은 자동 저장되어 다음 실행에 복원됩니다. 카메라/기타를 움직였다면 **다시 보정**하세요.

> 팁: 보정 중 **ESC**로 취소. 격자가 지판과 어긋나면 보정 4점이 잘못 찍힌 것이니 재보정하세요.
> `거울 모드`를 바꿔도 격자는 따라오지만, 정확도를 위해 셋업을 고정하는 것이 좋습니다.

### 🖐️ 운지 인식 (Phase 3)
보정이 끝나면 손끝(검지·중지·약지·새끼)이 지판 위 **몇 번 줄·몇 번 프렛**에 있는지 실시간으로 인식합니다.
- 지판 위 손끝마다 `1: 2번줄 1프렛`처럼 녹색 라벨이 붙습니다(`운지 라벨 표시` 토글).
- 우측 **운지 인식** 카드에 현재 짚고 있는 손가락이 줄 순서로 정리됩니다.
- 지판(보정한 1~K프렛, 6줄) 밖의 손끝(예: 스트로크 하는 손)은 자동으로 제외됩니다.
- 프렛 경계에서 값이 약간 떨릴 수 있습니다 → Phase 5에서 스무딩으로 안정화 예정.

### 🎼 코드 판정 (Phase 4)
**코드 판정** 카드에서 목표 코드를 고르면, 지금 잡은 운지를 실시간으로 채점합니다.
- `01 Source/chords/manifest.json`의 자료를 기준으로 메이저·마이너·7th·Major7·minor7·6th·sus4·7sus4 계열 60개 코드가 추가되었습니다.
- 기존 연습용 `Cadd9`까지 포함해 총 61개 목표 코드를 선택할 수 있습니다.
- 각 음을 **✅ 정확 / ⚠️ 위치·손가락 오류 / ❌ 빠짐**으로 분류하고 `정확도 2/3`처럼 점수를 표시.
- **교정 지시**를 구체적으로 안내: 예) "약지를 5번줄 2프렛 → 5번줄 3프렛으로 옮기세요", "검지로 2번줄 1프렛을 짚으세요", "3번줄은 개방현이니 새끼를 떼세요".
- `손가락 번호까지 검사 (엄격)` 체크 시: 위치가 맞아도 지정 손가락이 아니면 ⚠️로 표시(난이도↑). 끄면 위치만 맞으면 정답.
- 모든 음이 정확하고 불필요한 운지가 없으면 **✅ 정확합니다!**.

> 판정은 Phase 3의 손끝 인식을 그대로 쓰므로, 값이 떨리면 점수도 흔들릴 수 있습니다 → Phase 5에서 **홀드 게이트**(일정 시간 유지 시 확정)와 스무딩으로 안정화합니다.

### ✨ 교정 피드백 & 안정화 (Phase 5)
판정 결과를 **지판 위에 직접** 시각화하고, 떨림을 잡고, 음성으로도 코칭합니다.
- **색상 링**: 손끝에 정답=초록 / 위치·손가락 오류=노랑 / 불필요=빨강 링을 표시.
- **타깃 ◯ + 이동 화살표**: 짚어야 하는데 빈 음은 지판 위 **점선 ◯**(손가락 이름)로, 위치가 틀린 손가락은 현재→목표로 **화살표**를 그려 이동 방향을 안내.
- **손떨림 보정(스무딩)**: 각 손끝에 **One-Euro 필터**를 적용해 떨림을 줄입니다(토글, 기본 ON).
- **홀드 게이트**: 정답을 **0.7초** 연속 유지하면 **확정**(🔒) → **녹색 플래시 + 효과음**. 깜빡임으로 인한 오확정을 방지.
- **음성 코칭 🔊**(토글, 기본 OFF): 가장 중요한 교정 한 가지를 한국어로 읽어줍니다(쿨다운으로 과다 발화 방지). 정답 확정 시 "정확합니다".

> 이로써 **Phase 0~5 전체가 완성**되어, "웹캠으로 운지를 보고 → 코드가 맞는지 판정 → 어떻게 고칠지 시각·텍스트·음성으로 교정"하는 핵심 기능이 모두 동작합니다.

### 🔬 정밀 보정 & 코드/연주 설정 (Phase 6)
**정밀 보정**과 **카포·세븐스·바레 코드**를 추가했습니다.
- **보정 방식 선택**:
  - **빠름 (4점)** — 기존 방식. 지판 4모서리만 클릭.
  - **정밀 (프렛별)** — 너트~K프렛의 각 프렛선을 6번줄·1번줄 양 끝에서 클릭(총 2·(K+1)점). **최소제곱(overdetermined) homography**로 클릭 잡음을 평균화해 격자 정확도를 높입니다.
- **카포**: `카포 (프렛)` 값을 넣으면 목표 코드의 운지가 그만큼 위로 이동해 판정됩니다. (단, 보정 `K`가 카포+코드 최고 프렛을 덮어야 합니다 → 높은 카포는 `K`를 크게 보정하세요.)
- **코드 확장**: `01 Source/chords` 자료 기반으로 A~G 계열 메이저·마이너·7th·Major7·minor7·6th·sus4·7sus4, C#m/F#m 계열 마이너·minor7, Cadd9까지 추가.
  - 바레는 검지가 한 프렛의 여러 줄을 동시에 누르므로, 손끝 1점으로는 정확히 구분할 수 없어 **검지가 해당 프렛에 있으면 바레로 인정**하는 근사 판정을 씁니다. 교정 메시지: "검지로 N프렛 전체를 바레하세요".
  - C#m·C#m7·C7sus4처럼 6프렛까지 쓰는 코드가 있어 기본 **기준 프렛 K**는 `6`으로 조정했습니다.

> 검증: 최소제곱 homography가 무잡음 데이터를 정확 복원하고 n=4에서 4점 해와 일치, 바레(F)·카포 판정이 모두 통과(`ALL_PHASE6_TESTS_PASS`). 좌표·판정 핵심은 node 단위 테스트로 검증했습니다.

## 구성
| 파일 | 역할 |
|---|---|
| `index.html` | 화면 구조(비디오+오버레이 캔버스, 컨트롤 패널) |
| `style.css` | 다크 테마 스타일 |
| `main.js` | 카메라 제어 + MediaPipe Hand Landmarker + 캔버스 렌더 루프 |

## 기술 메모
- **MediaPipe Tasks Vision** `@0.10.35` (CDN, ES Module). 모델은 GPU 우선, 실패 시 CPU 폴백.
- **좌표 규약:** 거울 모드일 때 비디오는 CSS로 반전, 랜드마크는 `toDisplay()`가 x를 `(1-x)`로 뒤집어
  시각적으로 일치시킴. 캔버스는 반전하지 않아 텍스트가 정상.
- **캔버스 동기화:** 내부 해상도 = 실제 비디오 해상도. CSS가 표시 크기를 맞추므로 좌표가 어긋나지 않음.
- **손방향(handedness):** 카메라/미러 설정에 따라 Left/Right가 뒤바뀔 수 있습니다(표시는 raw 값).
  실제 운지 판정(Phase 4)에서는 손방향보다 위치가 중요하므로 큰 문제는 아닙니다.
